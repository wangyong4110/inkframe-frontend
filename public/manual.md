## 1. 平台简介

**简影**是一个 AI 驱动的"小说→视频"全栈创作平台。你可以：

- 用一句话描述，让 AI 自动生成百章长篇小说（多提供商 LLM + 层级上下文系统）
- 把小说章节一键转化为带配音、音效、BGM 的短视频草稿
- 导入已有小说，进行五级深度 AI 改写，生成符合版权合规要求的独立作品
- 管理角色、世界观、场景等创作资产，并复用于视频制作
- 在作品广场分享并发现其他创作者的作品

### 1.1 完整使用流程

#### 第一步：配置 AI 模型

首次使用前，前往顶部导航 **AI 模型** 页面，添加至少一个 AI 文本提供商（如 DeepSeek、OpenAI）并填写 API Key。这是所有 AI 功能的基础，跳过此步将无法生成任何内容。

> 详见 [10. AI 模型配置](#10-ai-模型配置)

#### 第二步：创建或导入小说

前往顶部导航 **创作台**，新建小说或导入已有内容：

- **AI 生成**：输入一段故事描述，设置类型和章节数，AI 自动规划大纲
- **导入**：粘贴文本、上传文件，或直接爬取起点/晋江等平台的章节

#### 第三步：逐章生成与编辑

大纲确认后，在小说详情页逐章操作：

1. 点击「生成章节」，AI 依据大纲和前文上下文写出完整章节内容
2. 在编辑器中修改文本、调整情节
3. 使用「AI 审查」功能检查逻辑一致性和文笔质量
4. 重复以上步骤，完成所有章节

#### 第四步：制作视频

章节完成后，在章节详情页点击「创建视频」：

1. **生成分镜**：AI 将章节内容自动拆解为若干镜头
2. **生成图像**：为每个分镜生成对应画面
3. **添加配音**：为镜头文本生成 AI 语音，支持多段配音和多种音色
4. **添加音效 / BGM**：AI 根据场景自动匹配音效和背景音乐
5. **导出**：导出为剪映草稿、Final Cut Pro 等格式进行后期制作

#### 第五步：发布与分享

视频制作完成后，可在**作品广场**发布，供其他用户浏览和点赞。也可通过素材库管理复用图片、音频等素材。

---

**快速上手路径（最短路径体验）：**

```
配置模型 → 创建小说（AI 生成） → 生成 1 章 → 创建视频 → 生成分镜 → 导出
```

### 1.2 核心技术亮点

| 技术 | 说明 |
|------|------|
| 层级上下文记忆 | 全局摘要 + 每 10 章弧线摘要 + 最近 2 章详文 + 最近 8 章简摘，避免长篇创作中的角色遗忘 |
| 多提供商 LLM | 同一任务可配置不同模型，大纲/正文/审查/分镜分别使用最合适的模型 |
| 场景一致性评分 | 每次图像生成后自动评分，绿/橙/红三色预警，保证视频画面风格统一 |
| 五级版权合规改写 | 从字词润色到深度蒸馏，量化词法/结构/语义相似度，帮助创作者规避版权风险 |
| 多段配音系统 | 单个镜头可配置多段独立语音，支持不同角色、不同提供商、不同音色混合 |

## 2. 注册与登录

### 2.1 注册账号

进入首页，点击右上角「注册」，填写邮箱和密码即可完成注册。注册后系统自动创建默认工作空间（租户），所有创作内容都属于该工作空间。

> 工作空间支持多成员协作。管理员可以在「工作空间设置」中邀请成员，设置不同角色（管理员/编辑/只读）。

### 2.2 登录

使用注册的邮箱和密码登录。登录后会看到**首页仪表盘**，包含：

- 最近创作的小说（快速续写入口）
- 平台热门视频推荐
- 热门小说推荐
- 近期任务执行状态摘要

### 2.3 个人资料

点击右上角头像 → **个人资料**，可修改：

- 昵称和头像
- 创作偏好设置（默认类型、默认语言）
- 通知偏好（任务完成时邮件提醒）

### 2.4 工作空间切换

如果你同时属于多个工作空间（如个人空间和团队空间），可以点击左上角工作空间名称进行切换。每个工作空间的小说、素材、AI 模型配置相互独立。

## 3. 创建小说

点击顶部导航「小说」→「创建小说」，有两种方式：

### 3.1 AI 智能生成

适合从零开始创作。

**填写基本信息：**

| 字段 | 说明 |
|------|------|
| 标题 | 小说名称，可留空让 AI 在大纲阶段自动命名 |
| 描述 | 核心故事设定，越详细越好。至少包含主角背景、核心冲突、基调情绪 |
| 类型 | 玄幻 / 仙侠 / 都市 / 言情 / 历史 / 悬疑 / 科幻 / 恐怖等 18 种 |
| 目标章节数 | 建议 10-100 章，超过 100 章时建议分卷管理 |
| 目标字数 | 每章目标字数，约 2000-3000 字为宜 |
| 创作语言 | 中文或英文。影响角色描述、物品描述、场景描述的生成语言，正文始终为中文 |
| 封面 | 上传图片，或点击「AI 生成封面」 |

**点击「生成大纲」后的过程：**

系统会按以下步骤异步执行（可在任务面板查看进度）：

1. AI 解析描述，确定核心主题、冲突弧线、人物关系
2. 规划若干"弧线"（每个弧线 10 章左右）
3. 为每章生成一句话提纲
4. 自动创建初始角色卡（主角、重要配角）
5. 生成基础世界观框架

> **最佳实践**：描述字段尽量写 3-5 句话，包含"主角是谁"、"最大的外部冲突"、"故事的情感核心"。例如："一个被逐出师门的炼丹师，在乱世中以毒攻毒重建家族荣耀，故事情感核心是师徒情谊与背叛。"

### 3.2 导入小说

适合基于已有内容继续创作或进行改写。

**三种导入方式：**

| 方式 | 说明 |
|------|------|
| 文件上传 | 上传 TXT / EPUB 文件，支持大文件分块上传，自动按章节切分 |
| 链接导入 | 粘贴小说网页 URL，系统自动抓取标题和内容 |
| 爬取 | 输入起点、晋江、纵横等平台的小说链接，自动批量抓取所有章节 |

导入完成后，系统会自动分析章节结构，你可以进一步在小说管理页面补充角色、世界观等设定。

## 4. 小说管理

点击小说列表中的任意一部小说，进入**小说详情页**。页面分为 7 个标签：章节、角色、物品、世界观、剧情点、场景、设置。

### 4.1 章节标签（章节）

这里是小说创作的核心区域。

**章节列表显示：**

| 信息 | 说明 |
|------|------|
| 章节编号和标题 | 按创作顺序排列 |
| 字数 | 实际生成的字符数 |
| 状态 | 草稿 / 生成中 / 已完成 |
| 质量评分 | 0-100 分，由四个维度综合计算 |
| 版本数 | 若多次生成或审查修改，会记录版本历史 |

**主要操作：**

- **生成下一章**：点击「生成下一章」按钮，AI 会参考大纲和前情摘要自动写作
- **批量生成**：设置连续生成的章节数（后台异步执行，可同时处理其他工作）
- **手动写章节**：点击「写作」进入章节编辑器，手动输入或修改内容
- **查看章节**：点击章节标题快速预览内容

**批量生成最佳实践：**

> 批量生成前，建议先手动生成并审查前 2-3 章，确认 AI 对风格和设定的理解符合预期，再开启批量模式。批量任务可在任务面板中暂停或取消。

### 4.2 角色标签（角色）

管理小说中出现的所有角色。

**添加角色：**

- **手动创建**：填写角色名、定位（主角/反派/配角/路人）、综合描述（包含外貌、性格、背景、能力等）
- **AI 批量生成**：描述故事背景，AI 自动生成若干角色设定，包含名称、定位和综合描述
- **从章节提取**：AI 扫描已完成章节，自动识别并创建出现的角色

**角色字段说明：**

| 字段 | 说明 |
|------|------|
| 名称 | 角色的中文名称（用于章节生成上下文） |
| 定位 | 主角 / 反派 / 配角 / 路人，影响在上下文中的权重 |
| 描述 | 统一的角色描述，包含外貌特征、性格特点、能力特长、背景故事等 |

**角色操作：**

| 操作 | 说明 |
|------|------|
| 生成三视图 | AI 生成正面/侧面/背面参考图，用于视频中的角色画面一致性 |
| 生成面部特写 | 生成适合视频配图的面部特写，突出角色神态 |
| 预览配音 | 输入一段台词，试听该角色绑定音色的配音效果 |
| 查看弧线 | 查看角色在各章节中的状态变化轨迹（修为/位置/心理状态等快照） |
| 编辑描述 | 随时更新角色描述，修改后对后续章节的 AI 生成立即生效 |

**角色状态快照：**

每次生成章节后，AI 会自动为章节涉及的角色记录一次状态快照（通过「同步角色状态」触发）。快照内容包括：当前能力等级、所在位置、心理状态、与其他角色的关系变化。这些快照是后续章节生成的重要上下文，能有效避免角色状态遗忘。

**角色弧线追踪：**

点击角色卡上的「弧线」，可以看到该角色在各章节中的状态时间轴。系统会在弧线摘要中自动记录重大事件（如突破修炼境界、与主角决裂等），并在每 10 章后生成一次弧线摘要，供后续章节生成参考。

> **最佳实践**：为主角和主要反派角色生成三视图后，视频制作时的图片一致性会显著提升。建议在开始大量章节生成前，先把主要角色的描述完善。

### 4.3 物品标签（物品）

管理小说中的重要道具、神器、法宝等。

**添加物品：**

- **手动创建**：填写物品名称和描述（包含外观、功能、来历、特殊能力等）
- **AI 提取**：点击「从章节提取」，AI 自动识别章节中出现的重要物品并创建条目
- **章节级覆盖**：可以为特定章节设置物品的阶段性描述（如某件神器在不同章节中形态不同）

**物品在章节生成中的作用：**

重要物品的描述会被注入到章节生成的上下文中，引导 AI 在提到该物品时保持描述一致性（外观、能力边界不发生矛盾）。

### 4.4 世界观标签（世界观）

构建小说的背景设定体系。

**创建世界观：**

- 填写世界观名称、描述、核心规则体系（修炼/魔法/科技体系）、地理设定
- 或点击「AI 生成世界观」，输入关键词（如"东方玄幻修炼世界，分金丹/元婴/化神境界"）自动生成

**世界观核心字段：**

| 字段 | 说明 |
|------|------|
| 世界名称 | 如"苍穹大陆"、"修真界" |
| 世界描述 | 整体背景介绍，AI 生成章节时作为基础上下文 |
| 力量体系 | 修炼/魔法/能力体系的等级和规则描述 |
| 地理设定 | 主要地域、国家、城市的分布描述 |
| 时代背景 | 历史背景、当前时代的主要势力格局 |

**添加世界观实体：**

点击「添加实体」，选择类型后填写：

| 类型 | 说明 |
|------|------|
| 地点 | 重要场景地点（秘境、宗门、城市等），影响场景生成 |
| 组织 | 宗门、家族、势力等，包含成员结构和目标 |
| 神器 | 重要传说道具，不同于物品的"神话级"设定 |
| 种族 | 异族、妖族等，包含文化和能力特征 |
| 生物 | 特殊生物和怪物，包含习性和危险等级 |

> 世界观信息越丰富，AI 生成的章节质量越高，角色行为越一致，世界细节越饱满。建议在开始大量章节生成前，至少完善力量体系和 3-5 个核心地点的描述。

### 4.5 剧情点标签（剧情点）

追踪小说中的伏笔、冲突和揭秘节点。

**添加剧情点：**

| 类型 | 说明 |
|------|------|
| 伏笔 | 早期埋下的线索，需要在后续章节中响应 |
| 冲突 | 正在进行的矛盾或对立，需要推进或解决 |
| 揭秘 | 计划在某章节揭露的秘密或反转 |

**操作：**

- **手动添加**：选择类型，描述剧情点内容和预计解决章节
- **AI 提取**：从已完成章节中自动识别未解决的伏笔和冲突
- **标记已解决**：点击剧情点旁的「解决」按钮，后续生成时不再追踪该剧情点

**剧情点在章节生成中的作用：**

大纲生成和章节生成时，系统会自动将**未解决的伏笔和冲突**注入上下文，引导 AI 合理推进或收束。当剧情点过多时，系统会按优先级排序，优先注入最紧迫的。

> **最佳实践**：在前 10 章中故意埋下 3-5 个伏笔，并在剧情点中记录，AI 会在合适的位置自然地回应这些伏笔，使故事更有整体感。

### 4.6 场景标签（场景）

管理小说中重要地点的视觉参考图，用于保证视频画面风格一致。

**场景 vs 世界观实体（地点）：**

世界观中的"地点"是文字描述层面的设定；场景标签则专注于**视觉参考**，关联具体的参考图片，用于指导图像生成保持一致性。

**操作说明：**

| 操作 | 说明 |
|------|------|
| AI 提取场景 | 从已完成章节的文本中自动识别出现的重要地点，批量创建场景条目 |
| 生成参考图 | 为场景描述生成视觉参考图（使用配置的图像生成提供商） |
| 锁定参考图 | 锁定后，后续该场景相关的视频分镜将以此图为一致性基准 |
| 手动上传 | 上传自己准备的参考图，替代 AI 生成的版本 |
| 查看一致性日志 | 查看所有使用过该场景参考图的分镜及评分历史 |

**一致性评分说明：**

每次生成分镜图片后，系统会自动与锁定的参考图进行多维度对比评分：

| 评分区间 | 颜色 | 含义 |
|----------|------|------|
| ≥ 85% | 绿色 | 通过，风格高度一致 |
| 70% - 85% | 橙色 | 建议重试，存在一定偏差 |
| < 70% | 红色 | 需要人工干预，风格差异明显 |

评分维度包括：建筑风格、光线氛围、色调、道具细节等。

**一致性评分最佳实践：**

1. 首先为小说中出现频率最高的 3-5 个场景生成并锁定参考图
2. 生成分镜图片时，系统会自动识别当前场景并参考锁定的图片
3. 对于红色评分的镜头，手动添加精修指令后重新生成

### 4.7 设置标签（设置）

修改小说基本信息。

**可修改的字段：**

| 字段 | 说明 |
|------|------|
| 标题 | 小说名称 |
| 简介 | 供读者看到的简介（与创作描述相互独立） |
| 类型 | 小说类型标签（可多选） |
| 目标字数 | 每章目标字数，影响后续 AI 生成长度 |
| 创作语言 | 中文/英文，影响角色、物品、场景描述的生成语言 |
| 可见性 | 公开 / 仅成员 / 私密 |
| 封面 | 更换封面图片 |

**关于创作语言（中文/英文）：**

- 选择「English」后，AI 生成的角色描述、物品描述、场景描述字段会使用英文
- 这在视频制作阶段非常有用——英文描述可以直接用于图像生成 Prompt，无需手动翻译
- 小说正文（章节内容）始终为中文，不受此设置影响

## 5. 章节编写

在小说详情页点击任意章节，或点击「写作」按钮，进入章节编辑器。

编辑器顶部有 5 种模式切换：大纲 / 写作 / 角色 / 场景 / 分镜脚本。

### 5.1 大纲模式

查看和编辑当前章节的创作提纲。

**大纲包含：**

- 本章核心目标（推进什么主线/支线）
- 场景分解（本章发生在哪里，几个场景）
- 节奏设计（开场基调、高潮点、结尾留白）
- 待解决的剧情点
- 本章的角色状态变化预期

**操作：**

- 直接编辑大纲文本，点击「保存大纲」
- 点击「按大纲生成正文」，AI 会严格按照你设定的场景大纲写作，不会自由发挥
- 点击「AI 优化大纲」，AI 根据前情上下文对当前大纲提出改进建议

### 5.2 写作模式

全屏文本编辑器，是日常修改和生成的主要界面。

**编辑器功能：**

- **直接编辑**：像 Word 一样修改章节内容，支持撤销/重做
- **字数统计**：实时显示章节字数
- **版本历史**：每次 AI 生成或手动保存都会创建版本快照，可在版本面板中查看和回滚

**AI 功能：**

| 功能 | 触发方式 | 说明 |
|------|----------|------|
| AI 续写 | 选中一段文字后，点击「AI 续写」 | AI 从所选位置继续生成后续内容 |
| AI 扩写 | 选中段落后，点击「AI 扩写」 | 将所选段落扩展为更详细的描写 |
| AI 深度审查 | 点击顶部「AI 深度审查」按钮 | 全面检查逻辑、一致性、文笔问题 |

**AI 深度审查详细说明：**

AI 深度审查是章节质量控制的核心工具。

**触发步骤：**

1. 在写作模式中，点击顶部「AI 深度审查」按钮
2. 系统会提交异步任务，通常 1-2 分钟完成
3. 审查完成后，右侧面板展示三个标签：当前审查 / 历史审查 / 忽略列表

**审查结果结构：**

审查结果以"问题列表"呈现，每条问题包含：

| 字段 | 说明 |
|------|------|
| 段落定位 | 具体是哪个段落出现了问题（段落高亮显示） |
| 问题类型 | 逻辑矛盾 / 角色不一致 / 文笔问题 / 风格偏差 |
| 严重程度 | 严重 / 中等 / 轻微（不同颜色区分） |
| 问题描述 | AI 解释为什么这是个问题 |
| 修改建议 | AI 给出的具体修改方案（可直接应用） |

**应用修改建议：**

- 点击单条问题的「应用」，AI 将对应段落按建议修改，并创建新版本
- 勾选多条问题后，点击「批量应用选中修改」，一次性处理多个问题
- 点击「忽略」，将该问题加入忽略列表，下次审查时不再提示

**忽略列表管理：**

审查面板的「忽略列表」标签显示所有已忽略的问题。可以随时取消忽略，恢复对该类问题的检测。

**历史审查记录：**

每次审查结果都被保存为历史记录，可以回溯到更早的审查结果版本，甚至基于早期审查记录回滚内容。

**质量评分维度：**

| 维度 | 权重 | 说明 |
|------|------|------|
| 情节逻辑 | 30% | 因果关系是否成立，行为动机是否合理 |
| 角色/世界观一致性 | 25% | 角色状态、能力边界是否与设定匹配 |
| 文笔质量 | 25% | 描写细腻度、语言节奏、用词水准 |
| 风格契合度 | 20% | 是否符合小说的整体基调和类型 |

### 5.3 角色模式

查看本章涉及角色的当前状态快照。

**显示内容：**

- 当前章节涉及的角色列表
- 每个角色在本章开始时的状态快照（来自上一章末的状态记录）
- 角色在本章中发生的主要变化（AI 预测或上次同步后记录）

**操作：**

- 点击「同步角色状态」，AI 会根据本章内容自动更新每个角色的状态记录
- 手动编辑某角色的当前状态快照
- 点击角色名称，跳转到该角色的详情页

> **注意**：同步角色状态建议在章节内容确认无误后执行，因为状态快照会影响后续章节的 AI 生成上下文。

### 5.4 场景模式

查看本章中出现的场景锚点，确认画面描述和视觉参考图。

**显示内容：**

- AI 自动识别本章中出现的场景地点
- 每个场景的视觉描述和锁定状态
- 关联的参考图（如有锁定）

**操作：**

- 手动关联场景与世界观地点
- 确认/修改场景描述
- 查看该场景的历史一致性评分

### 5.5 分镜脚本模式

直接从当前章节快速生成分镜脚本，跳转至视频制作。

**配置项：**

| 配置 | 说明 |
|------|------|
| 视频类型 | 图片解说（每镜一图+动效，成本低）或 AI 视频（逐帧视频，需视频 API） |
| 画面比例 | 16:9（横屏）/ 9:16（竖屏/短视频）/ 1:1（方形） |
| 节奏控制 | 慢 / 标准 / 快（影响分镜数量和单镜时长） |
| 目标时长 | 自动 / 1分钟 / 3分钟 / 5分钟 |

点击「生成分镜」，系统会创建视频项目并跳转至视频编辑器。

## 6. 视频制作

进入**视频编辑器**（从章节页创建，或在「视频」菜单中选择已有项目）。

编辑器包含以下标签：分镜脚本、配音、音效、BGM、时间线、合成导出。

### 6.1 分镜脚本标签

视频的骨架，每行对应一个镜头。

**分镜字段完整说明：**

| 字段 | 含义 | 备注 |
|------|------|------|
| 镜头编号 | 播放顺序（shot_no） | 自动维护，插入镜头时自动重排 |
| 画面描述 | 告诉 AI 画面中有什么 | **英文**，直接用于图像生成 Prompt |
| 旁白 | 播放该镜头时的旁白文案 | 中文，用于 TTS 语音合成 |
| 台词 | 角色说的话 | 有台词时，旁白留空，以台词为配音文本 |
| 时长 | 该镜头的画面时长（秒） | 影响配音截断和 BGM 分配 |
| 景别 | 特写 / 中景 / 远景 / 大远景 | 影响图像生成的构图风格 |
| 运镜 | 静止 / 推进 / 拉远 / 平摇 / 俯仰 | 用于动效生成和剪映草稿 |
| 转场 | 直切 / 淡入淡出 / 溶解 | 用于导出剪映草稿时的转场效果 |

**分镜节奏控制：**

生成分镜时，系统根据你选择的"节奏"和"目标时长"自动计算预期镜头数：

| 节奏 | 每分钟镜头数 | 单镜平均时长 |
|------|-------------|-------------|
| 慢 | 3-4 镜 | 15-20 秒 |
| 标准 | 5-8 镜 | 8-12 秒 |
| 快 | 9-15 镜 | 4-7 秒 |

**分镜图片操作：**

| 操作 | 说明 |
|------|------|
| 生成图片 | 点击单个镜头上的图片图标，为该镜头生成配图 |
| 批量生成图片 | 点击顶部「批量生成图片」，一次为所有镜头生成图 |
| 重新生成 | 对单个镜头图片不满意时，点击「重新生成」 |
| AI 精修 | 点击图片，在弹窗中输入修改指令（如"加强光影对比"、"改为夜景"） |
| 手动上传 | 点击镜头图片区域，上传自己准备的图片替代 AI 生成版本 |

**镜头管理操作：**

| 操作 | 触发方式 | 说明 |
|------|----------|------|
| 插入镜头 | 鼠标移至两个镜头之间，点击「+」 | 在指定位置插入空白镜头，后续镜头编号自动重排 |
| 复制镜头 | 点击镜头卡上的复制图标 | 在当前镜头之后复制一份，包含所有字段 |
| 删除镜头 | 点击删除图标 | 确认后删除，后续编号自动压缩 |
| 拖拽排序 | 拖动镜头卡（待实现） | — |

**AI 分镜审查：**

点击顶部「AI 审查分镜」，AI 会检查：

- 画面描述的一致性（是否与前后镜头场景冲突）
- 叙事节奏（是否某段过长或过短）
- 场景锚点一致性（画面中的环境是否与锁定参考图匹配）
- 角色视觉一致性（不同镜头中同一角色的描述是否一致）

审查结果支持应用修改建议和忽略问题，操作方式与章节审查相同。

### 6.2 配音标签

为视频添加旁白和角色台词配音。

**支持的 TTS 提供商：**

| 提供商 | 适合场景 | 代表音色 |
|--------|----------|----------|
| 阿里云 TTS | 中文旁白，音质自然流畅 | 龙小淳（女）、龙飞（男）、龙晓秀（轻快女）、龙晓白（沉稳男） |
| 百度 TTS | 多风格中文，情感丰富 | 度小美（女）、度逍遥（磁性男）、度丫丫（童声）、度小娇（温柔女） |
| MiniMax TTS | 情感细腻，适合戏剧性对白 | 少女音、御姐音、天美音、青年男声精英款、有声书专用系列 |
| 腾讯云 TTS | 专业播报，支持中英文 | 智言（男）、智雅（女）、智燕（暖女）、WeJack（英文男）、WeRose（英文女） |

**配音操作步骤：**

1. 在配音标签顶部选择提供商（下拉菜单）
2. 选择音色（不同提供商的音色列表会动态加载）
3. 设置语速（默认 1.0，范围 0.5-2.0，建议旁白 0.9、台词 1.0）
4. 点击「批量生成配音」，为所有镜头的旁白/台词生成语音
5. 点击单个镜头的播放按钮，试听配音效果
6. 不满意时，修改旁白文案或切换音色后点击「重新生成」

**多段配音（高级功能）：**

适用于同一镜头中多个角色交替说话的场景。

**展开多段配音步骤：**

1. 点击镜头卡上的「多段配音」展开按钮
2. 多段列表中，每段独立配置：文本内容、音色（可与全局音色不同）、语速
3. 在两段之间点击「插入」，在中间加入新段落
4. 每段独立生成、独立试听、独立删除
5. 最终合成时，多段按顺序拼接为该镜头的完整配音

> **多段配音使用场景**：角色 A 说了一句话，旁白补充一句，角色 B 回应一句。三段分别使用不同音色，最终无缝拼接。

**配音与时长同步：**

系统会根据配音时长自动调整分镜建议时长。若配音比设定时长更长，导出时以配音时长为准；若更短，以镜头设定时长为准（尾部静音填充）。

### 6.3 音效标签

为视频添加环境音和动作音效，增强沉浸感。

**自动生成流程：**

1. 点击「AI 分析音效标签」，AI 根据每个镜头的画面描述生成音效搜索关键词（英文标签）
2. 点击「批量生成音效」，系统使用配置的 SFX 提供商（如可灵文生音效）自动生成对应音效文件
3. 每个镜头下方展示生成的音效列表，点击播放图标试听
4. 启用/禁用单条音效（禁用后导出时不包含该音效）
5. 拖动时间范围滑块，设置音效在镜头中的起止时间

**手动操作：**

- 手动添加音效：输入描述或关键词，手动触发单条音效生成
- 从素材库选择：点击「从素材库选择」，选择已上传的音效文件
- 删除音效：点击音效条目右侧的删除图标

**音效生成技术说明（可灵文生音效）：**

- 使用可灵（Kling）文生音效 API，根据文本描述生成 3-10 秒音效
- 异步任务：提交后约 30-60 秒返回结果
- 生成失败时，系统会显示错误信息，可以重试或修改描述后再次生成

### 6.4 BGM 标签

为视频添加背景音乐，渲染情绪氛围。

**自动分配：**

1. 点击「AI 分析情绪」，AI 根据各镜头的旁白/台词内容判断情绪基调（热血/悲伤/神秘/轻松等）
2. 系统根据情绪标签从 Jamendo 免版权曲库中匹配合适的 BGM 片段
3. 匹配结果显示为时间段划分，每段对应一首 BGM

**手动调整：**

| 操作 | 说明 |
|------|------|
| 搜索替换 | 点击 BGM 段落，在搜索框输入关键词，从 Jamendo 曲库搜索替换 |
| 调节音量 | 拖动滑块，推荐范围 0.15-0.40（避免盖过旁白） |
| 调节范围 | 拖动时间段边界，调整这首 BGM 覆盖的镜头范围 |
| 启用/禁用 | 切换开关，禁用后导出时该段 BGM 静音 |

> **BGM 音量建议**：旁白场景建议 0.15-0.25，无旁白的纯画面场景可调高至 0.35-0.50。

### 6.5 时间线标签

可视化所有镜头的时间轴布局，用于预览整体节奏。

**显示内容：**

- 每个镜头的时长条（颜色区分景别）
- 配音轨道
- 音效轨道
- BGM 轨道
- 总时长估算

点击时间线上的任意位置，可以跳转到对应镜头的详情编辑。

### 6.6 合成与导出

**生成动效片段（图片解说模式）：**

点击「批量生成动效」，为每个镜头的静态图生成 Ken Burns 动效（推进/拉远/平移），生成后可拼接为完整视频。

> 注意：动效生成为 CPU 密集操作，单镜头约 30-60 秒，建议分批处理，或在后台运行时去做其他工作。

**导出格式：**

点击右上角「导出」，选择需要的格式：

| 格式 | 用途 | 推荐场景 |
|------|------|----------|
| 剪映草稿 | 导入剪映 App 进行二次剪辑 | 最推荐，保留完整的分镜、配音、BGM 结构 |
| B 剪草稿 | 供导演/剪辑参考的分镜注释包 | 团队协作 |
| Final Cut Pro XML | 导入 Final Cut Pro / DaVinci Resolve | 专业剪辑工作流 |
| 素材包（ZIP） | 图片/视频/音频原始文件打包 | 手动剪辑或存档备用 |
| SRT 字幕 | 通用字幕文件 | 上传到视频平台使用 |
| WebVTT 字幕 | 网络视频字幕 | 网页播放器 |
| 分镜表（CSV） | 可在 Excel / Notion / Airtable 中打开 | 团队沟通、剧本审核 |

**导入剪映步骤（macOS）：**

1. 点击「导出剪映草稿」，下载并解压 ZIP，得到一个文件夹
2. 将该文件夹整体（不是里面的文件）移入：
   ```
   ~/Movies/JianyingPro/User Data/Projects/com.lveditor.draft/
   ```
3. 重启剪映 → 草稿列表中会自动出现该项目
4. 双击打开，即可在剪映中进行二次剪辑

**导入剪映步骤（Windows）：**

1. 解压 ZIP
2. 将文件夹移入：
   ```
   C:\Users\{用户名}\AppData\Local\JianyingPro\User Data\Projects\com.lveditor.draft\
   ```
3. 重启剪映即可

**发布到外部平台：**

点击「发布」，选择目标平台（YouTube / 抖音 / Bilibili），填写标题、标签和描述，一键发布（需要提前在设置中授权对应平台账号）。

## 7. 小说改写

适合将已有版权小说改写为合规的独立原创作品，规避版权风险。本功能于 2026 年全面重设计，采用版权法视角的五级改写体系。

### 7.1 五级改写体系

| 级别 | 名称 | 改写深度 | 版权法定性 | 适用场景 |
|------|------|----------|-----------|----------|
| Level 1 | 字词润色 | 仅修改用词和短语，保留原有句式 | 衍生作品·需授权 | 个人练习，不适合发布 |
| Level 2 | 文学精炼 | 句子改写、节奏调整、文笔提升 | 衍生作品·需授权 | 个人学习，不适合发布 |
| Level 3 | 题材借鉴 | 保留核心主题和部分情节骨架 | 灰色地带 | 需谨慎，建议法律咨询 |
| Level 4 | 精神传承 | 仅继承精神内核，情节/人物/世界观全面重构 | **独立作品（推荐）** | 商业发布的推荐选择 |
| Level 5 | 深度蒸馏 | 完全重构，原作仅作为主题灵感 | **独立作品** | 最高合规度 |

> **法律免责说明**：以上定性仅为参考，具体版权合规需咨询专业法律人士。简影不承担因改写内容产生的版权纠纷责任。

**如何选择级别：**

- 目标是**商业发布**：选 Level 4 或 Level 5
- 目标是**平台发布/博主创作**：选 Level 3 或 Level 4
- 仅为**学习/研究**：任何级别均可

### 7.2 创建改写项目

进入「改写」菜单 → 「新建改写项目」：

1. 选择一部已导入的小说作为**原著**
2. 填写**改写项目名称**（改写后的作品名称）
3. 选择**改写级别**（Level 1-5，详见上表）
4. 点击「创建项目」

创建后进入改写项目详情页，分为三个阶段：分析原著 → 生成改写圣经 → 改写章节。

### 7.3 第一阶段：原著分析

点击「开始分析」（异步任务，通常 3-10 分钟）。

**分析内容：**

AI 会提取原著的以下信息：

| 分析维度 | 说明 |
|----------|------|
| 文学风格分析 | 作者的写作风格特征、常用修辞、句式模式 |
| 人物关系图谱 | 主要角色及其相互关系 |
| 世界观体系 | 原作的世界设定和规则体系 |
| 情节骨架 | 主要情节节点和因果链 |
| 意象体系 | 作者常用的意象和隐喻 |
| 标志性表达 | 原作特有的语言标记（禁用词组候选） |

### 7.4 第二阶段：改写圣经（世界改写圣经）

分析完成后，系统自动生成**改写圣经**（Rewrite Bible）。改写圣经是 AI 执行改写的核心指导文档，可以在生成后手动编辑。

**改写圣经包含以下部分：**

#### 风格转化方案

描述将原著的写作风格转化为新风格的具体方案。例如：将"金庸式古典武侠文风"转化为"现代都市玄幻口语化风格"。

#### 角色改名方案

| 原著角色名 | 改写后角色名 | 改名逻辑 |
|-----------|------------|----------|
| 张无忌 | 陈无极 | 保留"无"字寓意，更换姓氏 |
| 赵敏 | 林暮雨 | 完全重新命名，保留女性气质 |

#### 世界观转化方案

将原著世界观的核心元素映射到新的设定框架中，保持内在逻辑的同时改变所有具体标识。

#### 情节抽象提炼

将具体情节抽象为通用的叙事结构，再用新的具体情节重新实例化。

#### 禁止词组（禁止词组）

列出原著中的**标志性短语**，这些短语不得出现在改写版本中：

```
禁止词组示例：
- "乾坤大挪移"
- "七伤拳"  
- "光明顶"
- "[角色A]轻轻一笑，眼神复杂"
```

#### 签名对话改写指南（签名对话改写指南）

原著中标志性对白的改写方向：

| 原始对话模式 | 改写指南 |
|-------------|----------|
| "我张无忌从不..." | 改为角色使用新名字，并调整语气为更直接、现代 |
| 特定的告白表达方式 | 改变场景和表达路径，保留情感内核 |

#### 意象迁移表（意象迁移表）

原著中标志性意象到新意象的映射：

| 原始意象（original_symbol） | 新意象（new_symbol） |
|---------------------------|-------------------|
| 明教圣火 | 深渊之焰 |
| 峨嵋山 | 苍穹峰 |
| 武当派 | 玄武宗 |

> **最佳实践**：生成改写圣经后，花 10-15 分钟仔细审查禁止词组和意象迁移表，手动删除你认为泛化的条目（如普通汉字词组），添加你认为应该禁止但 AI 漏掉的原著标志性表达。

### 7.5 第三阶段：改写章节

改写圣经确认后，点击「开始改写」（异步任务，并发处理所有章节）。

**改写过程：**

每个章节的改写按以下步骤执行：

1. 读取原章节内容和改写圣经
2. 按选定级别执行改写（Level 4/5 会完全重写情节，仅保留情感弧）
3. 输出改写结果
4. 自动计算合规指标

**进度查看：**

改写任务面板中可以看到每个章节的改写状态：等待中 / 改写中 / 改写完成 / 改写失败。

### 7.6 查看改写结果

改写完成后，在改写项目详情页查看结果：

**对比视图：**

左右两栏分别显示原文和改写结果，差异高亮显示。可以对改写结果进行手动编辑。

**合规报告标签（合规报告）：**

这是改写项目的核心评估界面。

**总体评级卡：**

| 评级 | 条件 |
|------|------|
| 绿色（通过） | `passed_rate ≥ 80%` 且 `avg_lexical_similarity < 20%` |
| 红色（不通过） | `passed_rate < 60%` 或 `avg_lexical_similarity ≥ 35%` |
| 黄色（中间态） | 其他情况 |

**四项量化指标：**

| 指标 | 说明 | 绿色阈值 |
|------|------|---------|
| 词法相似度（avg_lexical_similarity） | 改写版本与原文的字词重叠比例 | < 20% |
| 结构相似度（structural_similarity） | 段落结构重叠比例 | < 30% |
| 语义泄漏率（semantic_leakage） | 原作特有实体名称在改写中出现的比例 | < 10% |
| 质量分（quality_score） | 改写文本的综合质量评估 | > 75 |

**章节详情表：**

每个章节单独显示以上四项指标，以及状态（通过/不通过/待改写）。

**导出报告：**

点击「导出到剪贴板」，将合规报告以 Markdown 格式复制，可粘贴到文档工具中存档。

### 7.7 重新改写与调整

对于合规报告中不通过的章节，可以：

1. 点击章节右侧的「重新改写」，使用相同圣经重新执行改写
2. 先修改改写圣经（加强禁止词组，更换意象），再点击「重新改写」
3. 直接在改写结果文本编辑器中手动修改不通过的部分

**调整后重新评分：**

手动编辑改写结果后，点击「重新评分」，系统会重新计算该章节的合规指标。

## 8. 素材库

独立的媒体资源管理区域，可在多个小说和视频项目之间复用图片、音频、视频素材。

### 8.1 上传素材

进入「素材库」→ 点击「上传」，支持以下文件类型：

| 类型 | 支持格式 |
|------|----------|
| 图片 | JPG / PNG / WebP / HEIC |
| 音频 | MP3 / WAV / AAC / OGG |
| 视频 | MP4 / MOV / WebM |

上传后可以：

- 添加手动标签（如：山景、古典音乐、战斗场景）
- 点击「AI 自动打标签」，AI 识别内容并自动添加语义标签
- 填写素材描述（用于语义搜索）

### 8.2 搜索素材

**关键词搜索：**

在搜索框输入关键词，按类型（图片/音频/视频）过滤。

**语义搜索：**

输入描述性文字（如"夕阳下的古代城墙"），系统根据语义匹配最相关的素材，无需精确关键词匹配。语义搜索由 AI 向量化引擎驱动，效果优于精确关键词搜索。

**过滤选项：**

- 按上传时间排序
- 按文件大小过滤
- 按标签筛选
- 仅显示公共库素材

### 8.3 版本管理

每个素材文件都支持版本管理：

1. 点击素材 → 「上传新版本」，选择替换文件
2. 历史版本保留，可以在「版本历史」中查看所有版本
3. 点击任意历史版本的「回滚」，将素材还原到该版本
4. 在视频项目中使用了该素材的地方，会提示是否同步到最新版本

### 8.4 分享到公共库

**申请分享步骤：**

1. 点击素材 → 「申请分享到公共库」
2. 确认素材的版权说明（仅可分享自有版权或免版权素材）
3. 提交审核

**审核通过后：**

- 素材在工作空间公共库中对所有成员可见
- 分享者在个人资料页会获得「贡献者」标记

### 8.5 批量爬取

从免版权素材平台批量抓取并入库：

**支持的来源：**

| 平台 | 素材类型 | 说明 |
|------|----------|------|
| Unsplash | 图片 | 高质量摄影图，CC0 协议 |
| Freesound | 音效 | 各类环境音效，Creative Commons |
| Pixabay | 图片 / 视频 | 免版权商业可用 |

**操作步骤：**

1. 点击「爬取任务」→「新建爬取任务」
2. 选择来源平台
3. 输入关键词（支持多个关键词，逗号分隔）
4. 设置最大数量（建议单次不超过 100 条）
5. 点击「开始爬取」，后台自动抓取并入库
6. 爬取完成后在素材库中可搜索到这些素材

## 11. 常见问题

### Q1：章节生成质量不高怎么办？

按优先级逐步尝试：

1. **完善世界观和角色设定**：AI 上下文越丰富，生成质量越高。至少完善力量体系、3-5 个核心地点、主角描述
2. **在大纲中为本章添加详细提纲**：明确本章目标、场景顺序、高潮节点
3. **切换到更强的 AI 模型**：在「AI 模型」配置中，为章节生成任务切换到 GPT-4o 或 Claude
4. **调低 Temperature 参数**：将章节生成的 Temperature 降至 0.7-0.75，减少随机性，提高一致性
5. **生成后使用 AI 深度审查**：找出具体问题并应用修改建议
6. **手动修改后再次生成**：将当前章节手动修改到满意程度，再继续下一章

### Q2：视频分镜图片风格不一致怎么办？

1. **锁定场景参考图**：进入小说详情 → 「场景」标签，为出现频率最高的 3-5 个场景生成并锁定参考图
2. **生成角色三视图**：在「角色」标签中，为主要角色生成三视图，视频生成时作为一致性参考
3. **检查一致性评分**：在视频编辑器的分镜标签中，查看每个镜头的一致性评分，对红色镜头重新生成
4. **统一图像生成模型**：确保所有镜头使用同一个图像生成提供商和模型，不要混用
5. **添加统一风格词**：在画面描述中加入统一的风格词（如 "ancient Chinese fantasy, cinematic lighting, 4K"）

### Q3：配音和画面时长不同步？

- 系统默认以**配音时长**为该镜头的实际播放时长
- 如果配音比设定时长更短，导出时镜头末尾会静音填充
- 如仍感觉不同步，可以：
  1. 点击单个镜头的配音「重新生成」（使用修改后的旁白文案）
  2. 在配音标签中调节语速（降低语速会延长配音时长）
  3. 在分镜脚本中手动修改该镜头的「时长」字段

### Q4：剪映打开后素材显示"离线"？

说明素材使用的 CDN 链接已过期，或网络无法访问到存储服务器。解决方法：

1. 使用「素材包（ZIP）」格式导出，获取本地化的图片和音频文件
2. 在剪映中对离线素材逐个点击「定位文件」，手动指向本地文件路径
3. 或直接删除离线素材并从本地重新导入

### Q5：大纲生成失败或一直转圈？

1. **检查 AI 模型配置**：进入「AI 模型」页面，确认大纲生成任务已分配模型，API Key 填写正确
2. **减少目标章节数**：超过 50 章的大纲生成可能触发超时，建议分批规划（先生成 30 章，完成后再追加）
3. **在任务面板查看错误详情**：任务面板（顶部导航的钟形图标）中可以看到具体的错误信息
4. **切换模型**：Claude 和 GPT-4 的 JSON 格式稳定性较好，可以为大纲任务切换到这些模型
5. **简化描述**：如果故事描述过于复杂，可以适当简化，避免 AI 因信息过多而产生结构混乱

### Q6：任务面板里有任务一直"运行中"？

- AI 文本生成任务通常 1-5 分钟内完成
- 图片生成通常 30-90 秒
- 视频生成（AI 视频模式）通常 5-15 分钟
- 小说改写（并发处理所有章节）通常 10-30 分钟

如超过以上时间仍未完成：

1. 刷新页面，在任务面板查看任务详情，确认是否有错误信息
2. 如显示"失败"但未自动重试，点击任务右侧的「重新执行」
3. 服务重启后，未完成的任务会自动恢复执行（状态保持在"运行中"）

### Q7：如何写中文小说但用英文 Prompt 生成分镜图片？

这是系统的默认行为，无需特殊设置：

- 正文和大纲：始终为中文
- 分镜的「画面描述」字段：AI 在生成分镜脚本时自动输出英文，直接用于图像生成
- 角色/物品/场景的描述：在「设置」标签中将「创作语言」改为「English」后，AI 生成这些字段时会使用英文，进一步提升图像生成质量

### Q8：改写后如何进一步提升合规度？

如果合规报告显示黄色或红色评级，按以下步骤提升：

1. **升级改写级别**：如当前是 Level 3，升级到 Level 4 重新改写
2. **强化改写圣经**：
   - 在禁止词组中手动添加更多原著标志性短语
   - 补充意象迁移表，确保所有原创意象都有对应的替换
   - 在角色改名方案中，将所有角色名全部替换（包括配角）
3. **逐章手动精修**：对语义泄漏率高的章节，手动替换剩余的原著特有实体名称
4. **重新评分**：手动修改后点击「重新评分」，确认指标已改善

### Q9：可以同时使用多个 AI 提供商吗？

可以，这正是推荐的使用方式：

- 为不同任务类型分配不同提供商：如大纲生成用 DeepSeek，AI 审查用 Claude，图片生成用可灵
- 多个提供商可以并行工作，互不干扰
- 在「任务配置」中，每种任务类型可以独立指定提供商和模型

### Q10：如何估算 API 费用？

**文本生成费用（以 DeepSeek 为例）：**

| 任务 | 估算 Token 消耗 | 参考费用（DeepSeek 单价） |
|------|----------------|--------------------------|
| 单章生成（2000字） | 约 3000 Token | 约 ¥0.003 |
| 100 章小说全部生成 | 约 30 万 Token | 约 ¥0.3 |
| AI 深度审查（单章） | 约 5000 Token | 约 ¥0.005 |

> 以上为粗略估算，实际消耗因上下文长度、层级摘要等因素有所差异。建议在提供商平台设置消费限额，避免超支。

**图片生成费用：**

以可灵为例，每张图片约 ¥0.05-0.10。一个 30 镜头的视频项目约 ¥1.5-3.0。

**视频生成费用：**

以可灵视频 API 为例，每 5 秒视频约 ¥0.5-1.0。单集视频（30 镜头 × 5 秒）约 ¥15-30。

### Q11：批量生成章节时出现角色遗忘怎么办？

简影使用**层级上下文记忆系统**来缓解长篇创作中的遗忘问题：

- 最近 2 章：完整内容注入上下文
- 最近 3-10 章：每章的简短摘要
- 每 10 章生成一次弧线摘要，记录角色状态和剧情重大节点
- 全局摘要：整部小说的核心设定

如仍出现遗忘，可以：

1. 确保角色描述和世界观设定在小说管理页面填写完整
2. 在出现遗忘的章节前，手动使用「同步角色状态」功能
3. 在该章节的大纲中，手动写入角色的当前状态提醒
4. 生成后使用 AI 审查，发现一致性问题后修复

### Q12：能否为视频中的角色设置固定音色？

可以。在小说管理 → 「角色」标签中：

1. 点击角色卡，进入角色详情
2. 在「配音设置」区域选择该角色的专属音色（提供商 + 音色 ID）
3. 保存后，视频配音阶段，含有该角色台词的镜头段落会自动使用对应音色
4. 在多段配音模式下，系统会自动将不同角色的台词分配到对应音色

> **提示**：建议为主角设置一个有辨识度的音色（如 MiniMax 的精英青年或御姐音），与旁白的音色明显区分，提升视频表现力。
