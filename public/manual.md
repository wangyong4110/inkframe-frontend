## 1. 平台简介

**简影**是一个 AI 驱动的"小说→视频"全栈创作平台。你可以：

- 用一句话描述，让 AI 自动生成百章长篇小说（多提供商 LLM + 层级上下文系统）
- 把小说章节一键转化为带配音、音效、BGM 的短视频草稿
- 导入已有小说，进行五级深度 AI 改写，生成符合版权合规要求的独立作品
- 管理角色（含形象设计、状态快照、声线绑定）、世界观、知识库、场景等创作资产
- 在作品广场分享并发现其他创作者的作品

### 1.1 完整使用流程

#### 第一步：配置 AI 模型

首次使用前，前往顶部导航 **模型** 页面，添加至少一个 AI 文本提供商（如 DeepSeek、OpenAI）并填写 API Key。这是所有 AI 功能的基础，跳过此步将无法生成任何内容。

> 详见 [10. AI 模型配置](#10-ai-模型配置)

#### 第二步：创建或导入小说

前往顶部导航 **项目**，新建小说或导入已有内容：

- **AI 生成**：输入一段故事描述，设置类型和章节数，AI 自动规划大纲
- **导入**：粘贴文本、上传文件，或直接爬取起点/晋江等平台的章节

#### 第三步：逐章生成与编辑

大纲确认后，在小说详情页逐章操作：

1. 点击「生成章节」，AI 依据大纲和前文上下文写出完整章节内容
2. 在编辑器中修改文本、调整情节
3. 使用「AI 深度审查」功能检查逻辑一致性和文笔质量
4. 重复以上步骤，完成所有章节

#### 第四步：制作视频

章节完成后，在章节详情页点击「创建视频」：

1. **生成分镜**：AI 将章节内容自动拆解为若干镜头
2. **生成图像**：为每个分镜生成对应画面（可高清放大或 AI 精修）
3. **添加配音**：为镜头文本生成 AI 语音，支持多段配音和多种音色
4. **添加音效 / BGM**：AI 根据场景自动匹配音效和背景音乐
5. **导出**：导出为剪映草稿、Premiere Pro 等格式进行后期制作

#### 第五步：发布与分享

视频制作完成后，可在**作品广场**发布，供其他用户浏览和点赞。也可通过素材库管理复用图片、音频等素材。

---

**快速上手路径（最短路径体验）：**

```
配置模型 → 创建小说（AI 生成） → 生成 1 章 → 创建视频 → 生成分镜 → 导出
```

### 1.2 核心技术亮点

| 技术 | 说明 |
|------|------|
| 层级上下文记忆 | 全局摘要 + 每 10 章弧线摘要 + 最近 2 章详文 + 最近 8 章简摘，避免长篇创作中的角色遗忘 |
| 多提供商 LLM | 同一任务可配置不同模型，大纲/正文/审查/分镜分别使用最合适的模型 |
| 场景一致性评分 | 每次图像生成后自动评分，绿/橙/红三色预警，保证视频画面风格统一 |
| 五级版权合规改写 | 从字词润色到深度蒸馏，量化词法/结构/语义相似度，帮助创作者规避版权风险 |
| 多段配音系统 | 单个镜头可配置多段独立语音，支持不同角色、不同提供商、不同音色混合 |
| 图片高清处理 | 算法放大（双三次插值 + 锐化，秒级）和 AI 放大（AI 重绘增强，质量更高）两种模式 |
| 伏笔管理系统 | 伏笔分级分类、超期预警、废弃警告、强化追踪、时间轴视图，结构化管理复杂伏笔网络 |
| 角色形象系统 | 多套外观版本（如"少年期/觉醒后/变装"），自动按章节范围激活，驱动视频图像一致性 |
| 知识库 | 语义向量搜索 + 关键词搜索双引擎，将小说特有设定注入生成上下文 |
| 协作编辑 | 实时 SSE 同步 + 编辑锁，防止多人同时编辑冲突 |

## 2. 注册与登录

### 2.1 注册方式

进入首页，点击右上角「注册」，支持三种方式：

| 方式 | 说明 |
|------|------|
| 邮箱 + 密码 | 填写邮箱和密码，系统发送验证邮件，点击链接完成验证 |
| 手机号 + 短信验证码 | 输入手机号，接收 SMS 验证码，完成快速注册 |
| OAuth 第三方 | 支持 Google 等 OAuth 提供商一键授权登录 |

注册后系统自动创建默认工作空间（租户），所有创作内容都属于该工作空间。

### 2.2 登录与账号安全

- **忘记密码**：首页登录页点击「忘记密码」，填写邮箱接收重置链接
- **修改密码**：登录后点击右上角头像 → 个人资料 → 修改密码
- **退出登录**：点击头像 → 退出，服务端同步使 Token 失效

### 2.3 工作空间管理

工作空间是所有资源（小说、视频、素材、AI 模型配置）的隔离单元。

- 进入 **租户管理** 页面，可以查看当前工作空间的成员列表
- 邀请成员：填写邮箱或用户 ID，设置角色（管理员 / 编辑 / 只读）
- 每个成员的 AI 配额和权限独立管理
- 可通过顶部导航切换不同工作空间（如个人空间与团队空间）

## 3. 创建小说

点击顶部导航「项目」→「创建小说」，有两种方式：

### 3.1 AI 智能生成

适合从零开始创作。

**填写基本信息：**

| 字段 | 说明 |
|------|------|
| 标题 | 小说名称，可留空让 AI 在大纲阶段自动命名 |
| 描述 | 核心故事设定，越详细越好。至少包含主角背景、核心冲突、基调情绪 |
| 类型 | 玄幻 / 仙侠 / 都市 / 言情 / 历史 / 悬疑 / 科幻 / 恐怖等 18 种 |
| 目标章节数 | 建议 10-100 章，超过 100 章时建议分卷管理 |
| 目标字数 | 每章目标字数，约 2000-3000 字为宜 |
| 创作语言 | 中文或英文。影响角色描述、物品描述、场景描述的生成语言，正文始终为中文 |
| 封面 | 上传图片，或点击「AI 生成封面」 |

**点击「生成大纲」后的过程：**

系统会按以下步骤异步执行（可在任务面板查看进度）：

1. AI 解析描述，确定核心主题、冲突弧线、人物关系
2. 规划若干"弧线"（每个弧线 10 章左右）
3. 为每章生成一句话提纲
4. 自动创建初始角色卡（主角、重要配角）
5. 生成基础世界观框架

> **最佳实践**：描述字段尽量写 3-5 句话，包含"主角是谁"、"最大的外部冲突"、"故事的情感核心"。例如："一个被逐出师门的炼丹师，在乱世中以毒攻毒重建家族荣耀，故事情感核心是师徒情谊与背叛。"

### 3.2 导入小说

适合基于已有内容继续创作或进行改写。

**三种导入方式：**

| 方式 | 说明 |
|------|------|
| 文件上传 | 上传 TXT / EPUB 文件，支持大文件分块上传，自动按章节切分 |
| 链接导入 | 粘贴小说网页 URL，系统自动抓取标题和内容 |
| 爬取 | 输入起点、晋江、纵横等平台的小说链接，自动批量抓取所有章节 |

导入完成后，系统会自动分析章节结构，你可以进一步在小说管理页面补充角色、世界观等设定。

## 4. 小说管理

点击小说列表中的任意一部小说，进入**小说详情页**。页面分为 9 个标签：章节、角色、物品、技能、世界观、知识库、场景、伏笔、团队、设置。

### 4.1 章节标签

这里是小说创作的核心区域。

**章节列表显示：**

| 信息 | 说明 |
|------|------|
| 章节编号和标题 | 按创作顺序排列 |
| 字数 | 实际生成的字符数 |
| 状态 | 草稿 / 生成中 / 已完成 |
| 质量评分 | 0-100 分，由情节逻辑、角色一致性、文笔质量、风格契合四个维度综合计算 |
| 版本数 | 若多次生成或审查修改，会记录版本历史 |

**主要操作：**

- **生成下一章**：AI 参考大纲和前情摘要自动写作
- **批量生成**：设置连续生成的章节数，后台异步执行
- **插入章节**：在指定位置插入新章节，后续章节编号自动重排
- **调整顺序**：拖拽排序章节（支持批量重排）
- **批量发布 / 批量 AI 审查 / 批量生成摘要**：选中多章后批量执行

**大纲综合审查：**

点击顶部「大纲综合审查」，AI 会横向审查所有章节大纲的一致性：

- 弧线节奏是否流畅
- 角色出场逻辑是否清晰
- 冲突推进是否合理
- 大纲质量综合评分和改进建议

### 4.2 角色标签

管理小说中出现的所有角色，支持丰富的角色信息管理。

**添加角色：**

- **手动创建**：填写角色名、定位、综合描述
- **AI 批量生成**：描述故事背景，AI 自动生成若干角色设定
- **从章节提取**：AI 扫描已完成章节，自动识别并创建出现的角色（含主次角色分离）

**角色字段说明：**

| 字段 | 说明 |
|------|------|
| 名称 | 角色的中文名称（用于章节生成上下文） |
| 定位 | 主角 / 反派 / 配角 / 路人，影响在上下文中的权重 |
| 性别 / 年龄 | 基本属性，用于辅助生成和筛选 |
| 描述 | 统一的综合描述字段，自由填写外貌特征、性格特点、能力特长、背景故事、与其他角色的关系等。AI 也会从此字段提取视觉提示词，越详细越好 |
| 内在冲突 | 驱动角色行为的内心恐惧或矛盾，用于深化角色复杂性 |
| 核心欲望 | 角色的核心动机或目标，是行动的根本驱动力 |
| 声线配置 | AI 提取的角色说话习惯：常用词组、禁用词、标志性句式，用于多段配音时自动识别台词 |
| 弧线设计 | 角色在各阶段的成长变化设计（JSON 格式，含目标章节范围） |

**角色操作：**

| 操作 | 说明 |
|------|------|
| 生成三视图 | AI 生成正面/侧面/背面参考图，用于视频图像一致性 |
| 生成面部特写 | 生成适合视频配图的面部特写 |
| 上传肖像 | 手动上传角色参考图 |
| 预览配音 | 输入台词，试听角色绑定音色的效果 |
| 提取声线 | AI 分析已完成章节中的角色台词，自动提取声线配置 |
| 分析视觉一致性 | 横向对比该角色在不同章节的视觉描述是否一致 |

**角色外观版本（Character Look）：**

一个角色可以拥有多套外观版本，适用于年龄变化、变装、觉醒等情节：

1. 点击角色卡 → 「外观版本」
2. 创建新版本（如"少年期"、"觉醒后"），填写有效章节范围
3. 每个版本可以独立生成参考图或上传参考图
4. 在视频分镜生成时，系统会根据当前章节号自动激活对应版本

**角色状态快照：**

每次生成章节后，AI 会自动为涉及角色记录状态快照（通过「同步角色状态」触发）。快照内容包括：当前能力等级、所在位置、心理状态、与其他角色的关系变化。这些快照是后续章节生成的重要上下文。

**角色弧线追踪：**

点击角色卡上的「弧线」，可以看到该角色在各章节中的状态时间轴。系统会在弧线摘要中自动记录重大事件，并在每 10 章后生成一次弧线摘要，供后续章节生成参考。

> **最佳实践**：为主角和主要反派角色生成三视图后，视频制作时的图片一致性会显著提升。建议在开始大量章节生成前，先把主要角色的描述完善。

### 4.3 物品标签

管理小说中的重要道具、神器、法宝等。

**添加物品：**

- **手动创建**：填写物品名称和描述（包含外观、功能、来历、特殊能力等）
- **AI 提取**：点击「从章节提取」，AI 自动识别章节中出现的重要物品并创建条目
- **批量生成图片**：为所有物品一次性生成参考图

**章节级覆盖：**

可以为特定章节设置物品的阶段性描述（如某件神器在不同章节中形态不同）。在章节编辑器「物品」模式中操作。

**物品在章节生成中的作用：**

重要物品的描述会被注入到章节生成的上下文中，引导 AI 在提到该物品时保持描述一致性（外观、能力边界不发生矛盾）。

### 4.4 技能标签

管理小说中角色的技能、法术、招式等。

**添加技能：**

- **手动创建**：填写技能名称和描述（效果、条件、副作用、视觉表现）
- **AI 批量生成**：描述故事背景，AI 根据世界观自动生成若干技能设定
- **批量删除**：选中多个技能批量移除

**技能特效图：**

点击技能右侧「生成特效图」，AI 根据技能描述生成视觉特效参考图，可用于视频分镜制作中的技能施放画面。

### 4.5 世界观标签

构建小说的背景设定体系。

**创建世界观：**

- 填写世界观名称、描述、核心规则体系（修炼/魔法/科技体系）、地理设定
- 或点击「AI 生成世界观」，输入关键词（如"东方玄幻修炼世界，分金丹/元婴/化神境界"）自动生成

**世界观核心字段：**

| 字段 | 说明 |
|------|------|
| 世界名称 | 如"苍穹大陆"、"修真界" |
| 世界描述 | 整体背景介绍，AI 生成章节时作为基础上下文 |
| 力量体系 | 修炼/魔法/能力体系的等级和规则描述 |
| 地理设定 | 主要地域、国家、城市的分布描述 |
| 时代背景 | 历史背景、当前时代的主要势力格局 |

**世界观实体：**

点击「添加实体」，选择类型后填写：

| 类型 | 说明 |
|------|------|
| 地点 | 重要场景地点（秘境、宗门、城市等），影响场景生成 |
| 组织 | 宗门、家族、势力等，包含成员结构和目标 |
| 神器 | 重要传说道具，不同于物品的"神话级"设定 |
| 种族 | 异族、妖族等，包含文化和能力特征 |
| 生物 | 特殊生物和怪物，包含习性和危险等级 |
| 词汇 | 世界特有的术语、名词、专有称谓 |

> 世界观信息越丰富，AI 生成的章节质量越高，角色行为越一致，世界细节越饱满。

### 4.6 知识库标签

小说专属的语义知识库，用于存储和检索小说特有的设定、剧情事实、写作技巧等。

**添加知识条目：**

- **手动创建**：填写标题、内容分类和正文
- **批量导入**：上传结构化文本批量入库
- **AI 提取剧情点**：AI 扫描已完成章节，自动识别重要事件和事实创建条目

**搜索方式：**

| 搜索类型 | 说明 |
|----------|------|
| 关键词搜索 | 精确匹配标题和内容中的关键词 |
| 语义搜索 | 输入描述性文字，向量引擎匹配语义相近的条目，无需精确关键词 |

**知识库在章节生成中的作用：**

生成章节时，系统会自动从知识库语义搜索与当前情节最相关的条目，注入提示词上下文，有效避免设定矛盾和"遗忘"已建立的世界细节。

### 4.7 场景标签

管理小说中重要地点的视觉参考图，用于保证视频画面风格一致。

**场景 vs 世界观实体（地点）：**

世界观中的"地点"是文字描述层面的设定；场景标签则专注于**视觉参考**，关联具体的参考图片，用于指导图像生成保持一致性。

**操作说明：**

| 操作 | 说明 |
|------|------|
| AI 提取场景 | 从已完成章节的文本中自动识别出现的重要地点，批量创建场景条目 |
| 生成参考图 | 为场景描述生成视觉参考图（使用配置的图像生成提供商） |
| AI 精修参考图 | 点击已有参考图，在灯箱中输入修改指令，重新生成调整后的版本 |
| 锁定参考图 | 锁定后，后续该场景相关的视频分镜将以此图为一致性基准 |
| 手动上传 | 上传自己准备的参考图，替代 AI 生成的版本 |
| 高清放大 | 在灯箱中点击「高清处理」，对参考图进行 2× 超分辨率放大 |
| 查看一致性日志 | 查看所有使用过该场景参考图的分镜及评分历史 |

**一致性评分说明：**

每次生成分镜图片后，系统会自动与锁定的参考图进行多维度对比评分：

| 评分区间 | 颜色 | 含义 |
|----------|------|------|
| ≥ 85% | 绿色 | 通过，风格高度一致 |
| 70% - 85% | 橙色 | 建议重试，存在一定偏差 |
| < 70% | 红色 | 需要人工干预，风格差异明显 |

评分维度：建筑风格、光线氛围、色调、道具细节等。

### 4.8 伏笔标签

专业的伏笔全生命周期管理系统，追踪小说中所有埋下的线索和预设的揭示节点。

**伏笔字段说明：**

| 字段 | 说明 |
|------|------|
| 标题 | 伏笔的简短名称（如"神秘玉佩"、"导师的真实身份"） |
| 描述 | 伏笔的完整内容和埋设方式 |
| 级别 | **主线**（核心伏笔，必须精心回收）/ **支线**（重要伏笔，需明确安排）/ **细节**（小伏笔，可灵活处理） |
| 类型 | 道具伏线 / 对话伏线 / 行为伏线 / 场景伏线 / 预言 / 悬念钩 / 关系伏线 / 世界观伏线 |
| 埋设章节 | 伏笔首次出现的章节编号 |
| 预计回收章节 | 计划揭示或解决的目标章节 |
| 实际回收内容 | 填写后标记为"已回收"，记录实际揭示的内容 |
| 重要程度 | 关键 / 重要 / 一般 |
| 状态 | 已埋设 / 成熟中 / 已回收 / 已废弃 |
| 强化章节 | 记录在哪些章节中有意重复提及了该伏笔，加深读者印象 |
| 关联角色 | 该伏笔涉及的角色列表 |
| 关联钩子 | 关联到对应的故事钩子（悬念节点） |

**伏笔操作：**

- **手动添加**：选择类型和级别，描述伏笔内容，填写埋设和预计回收章节
- **AI 提取**：从已完成章节中自动识别未解决的伏笔
- **添加强化**：记录在某章节中对伏笔的重复提及
- **标记已回收**：填写「实际回收内容」并确认，系统记录回收质量（1-5 分）
- **时间轴视图**：切换到「时间轴」模式，按章节顺序可视化伏笔的完整生命轨迹

**预警机制：**

| 预警类型 | 触发条件 | 建议处理 |
|----------|----------|---------|
| 超期预警（橙色） | 当前章节超出预计回收章节 | 尽快安排回收，或修改预计章节 |
| 废弃警告（红色） | 伏笔超期过久且无回收迹象 | 在章节中补充衔接，或手动标记废弃 |

**伏笔统计：**

「统计」视图显示：已埋设 / 成熟中 / 已回收 / 已废弃的数量分布，以及平均回收质量评分，帮助你评估故事整体的伏笔健康度。

**伏笔树结构：**

复杂伏笔可以建立父子关系（如"主谋身份"伏笔下挂若干"线索碎片"子伏笔），「树视图」展示层级关系。

**伏笔在章节生成中的作用：**

生成章节时，系统会自动将**未回收的主线和支线伏笔**注入上下文，引导 AI 在合适的位置推进或收束。超期伏笔会被优先注入，提醒 AI 安排回收。

> **最佳实践**：在前 10 章中故意埋下 3-5 个伏笔，分别标注级别，主线伏笔填写精确的预计回收章节（设超期预警），细节伏笔可不设预计章节。

### 4.9 团队标签

管理小说的协作成员。

**功能：**

- 查看当前所有成员及角色（管理员 / 编辑 / 只读）
- 邀请成员：填写邮箱发送邀请链接
- 调整成员角色
- 移除成员
- 实时协作：多人同时在线时，系统通过 SSE 实时同步编辑状态；同一章节同时只允许一人编辑（编辑锁保护）

### 4.10 设置标签

修改小说基本信息。

**可修改的字段：**

| 字段 | 说明 |
|------|------|
| 标题 | 小说名称 |
| 简介 | 供读者看到的简介（与创作描述相互独立） |
| 类型 | 小说类型标签（可多选） |
| 目标字数 | 每章目标字数，影响后续 AI 生成长度 |
| 创作语言 | 中文/英文，影响角色、物品、场景描述的生成语言 |
| 可见性 | 公开 / 仅成员 / 私密 |
| 封面 | 更换封面图片 |

**关于创作语言（中文/英文）：**

- 选择「English」后，AI 生成的角色描述、物品描述、场景描述字段会使用英文
- 这在视频制作阶段非常有用——英文描述可以直接用于图像生成 Prompt，无需手动翻译
- 小说正文（章节内容）始终为中文，不受此设置影响

## 5. 章节编写

在小说详情页点击任意章节，或点击「写作」按钮，进入章节编辑器。

编辑器顶部有 5 种模式切换：大纲 / 写作 / 角色 / 场景 / 分镜脚本。

### 5.1 大纲模式

查看和编辑当前章节的创作提纲。

**大纲包含：**

- 本章核心目标（推进什么主线/支线）
- 场景分解（本章发生在哪里，几个场景）
- 节奏设计（开场基调、高潮点、结尾留白）
- 待解决的剧情点和伏笔
- 本章的角色状态变化预期

**操作：**

- 直接编辑大纲文本，点击「保存大纲」
- 点击「按大纲生成正文」，AI 会严格按照你设定的场景大纲写作
- 点击「AI 优化大纲」，AI 根据前情上下文对当前大纲提出改进建议
- 点击「大纲审查」，AI 对本章大纲的逻辑性和完整性进行评估

### 5.2 写作模式

全屏文本编辑器，是日常修改和生成的主要界面。

**编辑器功能：**

- **直接编辑**：像 Word 一样修改章节内容，支持撤销/重做
- **字数统计**：实时显示章节字数
- **版本历史**：每次 AI 生成或手动保存都会创建版本快照，可在版本面板中查看和回滚

**AI 功能：**

| 功能 | 触发方式 | 说明 |
|------|----------|------|
| AI 续写 | 选中一段文字后，点击「AI 续写」 | AI 从所选位置继续生成后续内容 |
| AI 扩写 | 选中段落后，点击「AI 扩写」 | 将所选段落扩展为更详细的描写 |
| AI 按指令改写 | 点击「按指令改写」，输入具体要求 | 如"改为第一人称"、"加强环境描写" |
| AI 精修选段 | 选中段落，点击「精修」 | 针对选中文字进行措辞和风格优化 |
| AI 与章节对话 | 点击「AI 对话」进入流式对话模式 | 以聊天方式与 AI 讨论章节问题、情节走向 |
| AI 深度审查 | 点击顶部「AI 深度审查」按钮 | 全面检查逻辑、一致性、文笔问题 |

**AI 深度审查详细说明：**

AI 深度审查是章节质量控制的核心工具。

**触发步骤：**

1. 在写作模式中，点击顶部「AI 深度审查」按钮
2. 系统会提交异步任务，通常 1-2 分钟完成
3. 审查完成后，右侧面板展示三个标签：当前审查 / 历史审查 / 忽略列表

**审查结果结构：**

审查结果以"问题列表"呈现，每条问题包含：

| 字段 | 说明 |
|------|------|
| 段落定位 | 具体是哪个段落出现了问题（段落高亮显示） |
| 问题类型 | 逻辑矛盾 / 角色不一致 / 文笔问题 / 风格偏差 |
| 严重程度 | 严重 / 中等 / 轻微（不同颜色区分） |
| 问题描述 | AI 解释为什么这是个问题 |
| 修改建议 | AI 给出的具体修改方案（可直接应用） |

**应用修改建议：**

- 点击单条问题的「应用」，AI 将对应段落按建议修改，并创建新版本
- 勾选多条问题后，点击「批量应用选中修改」，一次性处理多个问题
- 点击「忽略」，将该问题加入忽略列表，下次审查时不再提示
- 对某次历史审查结果不满意时，可点击「回滚」还原到审查前的版本

**质量评分维度：**

| 维度 | 权重 | 说明 |
|------|------|------|
| 情节逻辑 | 30% | 因果关系是否成立，行为动机是否合理 |
| 角色/世界观一致性 | 25% | 角色状态、能力边界是否与设定匹配 |
| 文笔质量 | 25% | 描写细腻度、语言节奏、用词水准 |
| 风格契合度 | 20% | 是否符合小说的整体基调和类型 |

**连贯性检查：**

除 AI 审查外，还有自动化的**连贯性检查**（Continuity Check）：
- 角色状态连贯性（能力是否超出已设定范围）
- 世界观规则遵守（魔法/修炼规则是否自洽）
- 时间线一致性（地点切换、时间流逝是否合理）

在「连贯性报告」标签中查看结果，支持按严重程度过滤。

### 5.3 角色模式

查看本章涉及角色的当前状态快照。

**显示内容：**

- 当前章节涉及的角色列表
- 每个角色在本章开始时的状态快照
- 角色在本章中发生的主要变化（AI 预测或上次同步后记录）

**操作：**

- 点击「同步角色状态」，AI 会根据本章内容自动更新每个角色的状态记录
- 手动编辑某角色的当前状态快照
- 点击角色名称，跳转到该角色的详情页

> **注意**：同步角色状态建议在章节内容确认无误后执行，因为状态快照会影响后续章节的 AI 生成上下文。

### 5.4 场景模式

查看本章中出现的场景锚点，确认画面描述和视觉参考图。

**显示内容：**

- AI 自动识别本章中出现的场景地点
- 每个场景的视觉描述和锁定状态
- 关联的参考图（如有锁定）

**操作：**

- 手动关联场景与世界观地点
- 确认/修改场景描述
- 查看该场景的历史一致性评分

### 5.5 分镜脚本模式

直接从当前章节快速生成分镜脚本，跳转至视频制作。

**配置项：**

| 配置 | 说明 |
|------|------|
| 视频类型 | 图片解说（每镜一图+动效，成本低）或 AI 视频（逐帧视频，需视频 API） |
| 画面比例 | 16:9（横屏）/ 9:16（竖屏/短视频）/ 1:1（方形） |
| 节奏控制 | 慢 / 标准 / 快（影响分镜数量和单镜时长） |
| 目标时长 | 自动 / 1分钟 / 3分钟 / 5分钟 |

点击「生成分镜」，系统会创建视频项目并跳转至视频编辑器。

## 6. 视频制作

进入**视频编辑器**（从章节页创建，或在「项目」→「视频」菜单中选择已有项目）。

编辑器包含以下标签：分镜脚本、配音、音效、BGM、时间线、合成导出。

### 6.1 分镜脚本标签

视频的骨架，每行对应一个镜头。

**分镜字段完整说明：**

| 字段 | 含义 | 备注 |
|------|------|------|
| 镜头编号 | 播放顺序（shot_no） | 自动维护，插入镜头时自动重排 |
| 画面描述 | 告诉 AI 画面中有什么 | **英文**，直接用于图像生成 Prompt |
| 旁白 | 播放该镜头时的旁白文案 | 中文，用于 TTS 语音合成 |
| 台词 | 角色说的话 | 有台词时，旁白留空，以台词为配音文本 |
| 时长 | 该镜头的画面时长（秒） | 影响配音截断和 BGM 分配 |
| 景别 | 特写 / 中景 / 远景 / 大远景 | 影响图像生成的构图风格 |
| 运镜 | 静止 / 推进 / 拉远 / 平摇 / 俯仰 | 用于动效生成和剪映草稿 |
| 转场 | 直切 / 淡入淡出 / 溶解 | 用于导出剪映草稿时的转场效果 |

**分镜节奏控制：**

生成分镜时，系统根据你选择的"节奏"和"目标时长"自动计算预期镜头数：

| 节奏 | 每分钟镜头数 | 单镜平均时长 |
|------|-------------|-------------|
| 慢 | 3-4 镜 | 15-20 秒 |
| 标准 | 5-8 镜 | 8-12 秒 |
| 快 | 9-15 镜 | 4-7 秒 |

**分镜图片操作：**

| 操作 | 说明 |
|------|------|
| 生成图片 | 点击单个镜头上的图片图标，为该镜头生成配图 |
| 批量生成图片 | 点击顶部「批量生成图片」，一次为所有镜头生成图 |
| 重新生成 | 对单个镜头图片不满意时，点击「重新生成」 |
| AI 精修 | 点击图片打开灯箱，在下方文本框输入修改指令（如"加强光影对比"、"改为夜景"），点击「重新生成」 |
| 算法高清放大 | 在灯箱底部点击「高清处理」，2× 双三次插值放大 + 自动锐化，秒级完成，无需消耗 AI 配额 |
| AI 高清放大 | 在灯箱底部点击「高清处理 ▼」→ 选「AI 放大」，AI 重绘方式 2× 增强，画面细节更丰富（消耗图像 API 配额，约 1-2 分钟） |
| 手动上传 | 点击镜头图片区域，上传自己准备的图片 |

> **高清放大说明**：算法放大适合快速出图使用，AI 放大适合对最终定稿画面进行质量提升。两者均通过后台任务异步执行，完成后在灯箱中可选择保存或放弃新图。

**镜头管理操作：**

| 操作 | 触发方式 | 说明 |
|------|----------|------|
| 插入镜头 | 鼠标移至两个镜头之间，点击「+」 | 在指定位置插入空白镜头，后续镜头编号自动重排 |
| 复制镜头 | 点击镜头卡上的复制图标 | 在当前镜头之后复制一份，包含所有字段 |
| 删除镜头 | 点击删除图标 | 确认后删除，后续编号自动压缩 |
| 绑定场景 | 为镜头指定对应的场景锚点 | 系统以该锚点的参考图作为一致性基准 |
| 绑定角色 | 为镜头指定出场角色 | 角色三视图作为人物形象一致性参考 |

**AI 分镜审查：**

点击顶部「AI 审查分镜」，AI 会检查：

- 画面描述的一致性（是否与前后镜头场景冲突）
- 叙事节奏（是否某段过长或过短）
- 场景锚点一致性（画面中的环境是否与锁定参考图匹配）
- 角色视觉一致性（不同镜头中同一角色的描述是否一致）

审查结果支持应用修改建议、批量应用插入/删除镜头建议，以及忽略问题，操作方式与章节审查相同。

### 6.2 配音标签

为视频添加旁白和角色台词配音。

**支持的 TTS 提供商：**

| 提供商 | 适合场景 | 代表音色 |
|--------|----------|----------|
| 阿里云 TTS | 中文旁白，音质自然流畅 | 龙小淳（女）、龙飞（男）、龙晓秀（轻快女）、龙晓白（沉稳男） |
| 百度 TTS | 多风格中文，情感丰富 | 度小美（女）、度逍遥（磁性男）、度丫丫（童声）、度小娇（温柔女） |
| MiniMax TTS | 情感细腻，适合戏剧性对白 | 少女音、御姐音、天美音、青年男声精英款、有声书专用系列 |
| 腾讯云 TTS | 专业播报，支持中英文 | 智言（男）、智雅（女）、智燕（暖女）、WeJack（英文男）、WeRose（英文女） |

**配音操作步骤：**

1. 在配音标签顶部选择提供商（下拉菜单）
2. 选择音色（不同提供商的音色列表会动态加载）
3. 设置语速（默认 1.0，范围 0.5-2.0，建议旁白 0.9、台词 1.0）
4. 点击「批量生成配音」，为所有镜头的旁白/台词生成语音
5. 点击单个镜头的播放按钮，试听配音效果
6. 不满意时，修改旁白文案或切换音色后点击「重新生成」

**多段配音（高级功能）：**

适用于同一镜头中多个角色交替说话的场景。

**展开多段配音步骤：**

1. 点击镜头卡上的「多段配音」展开按钮
2. 多段列表中，每段独立配置：文本内容、音色（可与全局音色不同）、语速
3. 在两段之间点击「插入」，在中间加入新段落
4. 每段独立生成、独立试听、独立删除
5. 最终合成时，多段按顺序拼接为该镜头的完整配音

> **多段配音使用场景**：角色 A 说了一句话，旁白补充一句，角色 B 回应一句。三段分别使用不同音色，最终无缝拼接。

**配音与时长同步：**

系统会根据配音时长自动调整分镜建议时长。若配音比设定时长更长，导出时以配音时长为准；若更短，以镜头设定时长为准（尾部静音填充）。

### 6.3 音效标签

为视频添加环境音和动作音效，增强沉浸感。

**音效类型：**

| 类型 | 说明 | 示例 |
|------|------|------|
| 动作音效 | 瞬间动作产生的声音 | 脚步声、打击音、爆炸 |
| 环境音效 | 场景持续的背景声 | 风声、市集喧嚣、流水 |
| 情绪音效 | 增强情感氛围的音乐短片段 | 悬疑音效、胜利号角 |

**自动生成流程：**

1. 点击「AI 分析音效标签」，AI 根据每个镜头的画面描述生成音效搜索关键词（英文标签）
2. 点击「批量生成音效」，系统使用配置的 SFX 提供商（如可灵文生音效）自动生成对应音效文件
3. 每个镜头下方展示生成的音效列表，点击播放图标试听
4. 启用/禁用单条音效（禁用后导出时不包含该音效）

**音效精细控制：**

| 控制项 | 说明 |
|--------|------|
| 音量 | 每条音效独立设置音量（0-200%） |
| 起止时间 | 拖动时间范围滑块，设置音效在镜头中的起止位置 |
| 是否循环 | 环境音效通常需要循环播放 |
| 淡入/淡出 | 设置音效的渐入和渐出时长（毫秒） |

**手动操作：**

- 手动添加音效：输入描述或关键词，手动触发单条音效生成
- 从素材库选择：点击「从素材库选择」，选择已上传的音效文件
- 删除音效：点击音效条目右侧的删除图标

### 6.4 BGM 标签

为视频添加背景音乐，渲染情绪氛围。

**自动分配：**

1. 点击「AI 分析情绪」，AI 根据各镜头的旁白/台词内容判断情绪基调（热血/悲伤/神秘/轻松等）
2. 系统根据情绪标签从 Jamendo 免版权曲库中匹配合适的 BGM 片段
3. 匹配结果显示为时间段划分，每段对应一首 BGM

**手动调整：**

| 操作 | 说明 |
|------|------|
| 搜索替换 | 点击 BGM 段落，在搜索框输入关键词，从 Jamendo 曲库搜索替换 |
| 调节音量 | 拖动滑块，推荐范围 0.15-0.40（避免盖过旁白） |
| 调节范围 | 拖动时间段边界，调整这首 BGM 覆盖的镜头范围 |
| 启用/禁用 | 切换开关，禁用后导出时该段 BGM 静音 |
| 音频闪避 | 配置有旁白时 BGM 自动降低的目标音量（默认 0.15） |

> **BGM 音量建议**：旁白场景建议 0.15-0.25，无旁白的纯画面场景可调高至 0.35-0.50。

### 6.5 时间线标签

可视化所有镜头的时间轴布局，用于预览整体节奏。

**显示内容：**

- 每个镜头的时长条（颜色区分景别）
- 配音轨道
- 音效轨道（含多条并行音效）
- BGM 轨道
- 总时长估算

点击时间线上的任意位置，可以跳转到对应镜头的详情编辑。

### 6.6 合成与导出

**生成动效片段（图片解说模式）：**

点击「批量生成动效」，为每个镜头的静态图生成 Ken Burns 动效（推进/拉远/平移），生成后可拼接为完整视频。

> 注意：动效生成为 CPU 密集操作，单镜头约 30-60 秒，建议分批处理。

**导出格式：**

点击右上角「导出」，选择需要的格式：

| 格式 | 用途 | 推荐场景 |
|------|------|----------|
| 剪映草稿 | 导入剪映 App 进行二次剪辑 | 最推荐，保留完整的分镜、配音、BGM 结构 |
| Premiere Pro XML | 导入 Adobe Premiere Pro / DaVinci Resolve | 专业剪辑工作流 |
| 素材包（ZIP） | 图片/视频/音频原始文件打包 | 手动剪辑或存档备用 |
| SRT 字幕 | 通用字幕文件 | 上传到视频平台使用 |
| WebVTT 字幕 | 网络视频字幕 | 网页播放器 |
| ASS 字幕 | 高级字幕（支持特效/样式） | 动漫字幕制作 |

**导入剪映步骤（macOS）：**

1. 点击「导出剪映草稿」，下载并解压 ZIP，得到一个文件夹
2. 将该文件夹整体（不是里面的文件）移入：
   ```
   ~/Movies/JianyingPro/User Data/Projects/com.lveditor.draft/
   ```
3. 重启剪映 → 草稿列表中会自动出现该项目
4. 双击打开，即可在剪映中进行二次剪辑

**导入剪映步骤（Windows）：**

1. 解压 ZIP
2. 将文件夹移入：
   ```
   C:\Users\{用户名}\AppData\Local\JianyingPro\User Data\Projects\com.lveditor.draft\
   ```
3. 重启剪映即可

## 7. 小说改写

适合将已有版权小说改写为合规的独立原创作品，规避版权风险。

### 7.1 五级改写体系

| 级别 | 名称 | 改写深度 | 版权法定性 | 适用场景 |
|------|------|----------|-----------|----------|
| Level 1 | 字词润色 | 仅修改用词和短语，保留原有句式 | 衍生作品·需授权 | 个人练习，不适合发布 |
| Level 2 | 文学精炼 | 句子改写、节奏调整、文笔提升 | 衍生作品·需授权 | 个人学习，不适合发布 |
| Level 3 | 题材借鉴 | 保留核心主题和部分情节骨架 | 灰色地带 | 需谨慎，建议法律咨询 |
| Level 4 | 精神传承 | 仅继承精神内核，情节/人物/世界观全面重构 | **独立作品（推荐）** | 商业发布的推荐选择 |
| Level 5 | 深度蒸馏 | 完全重构，原作仅作为主题灵感 | **独立作品** | 最高合规度 |

> **法律免责说明**：以上定性仅为参考，具体版权合规需咨询专业法律人士。简影不承担因改写内容产生的版权纠纷责任。

### 7.2 创建改写项目

进入「改写」菜单 → 「新建改写项目」：

1. 选择一部已导入的小说作为**原著**
2. 填写**改写项目名称**（改写后的作品名称）
3. 选择**改写级别**（Level 1-5）
4. 点击「创建项目」

创建后进入改写项目详情页，分为三个阶段：分析原著 → 生成改写圣经 → 改写章节。

### 7.3 第一阶段：原著分析

点击「开始分析」（异步任务，通常 3-10 分钟）。

AI 会提取原著的以下信息：

| 分析维度 | 说明 |
|----------|------|
| 文学风格分析 | 作者的写作风格特征、常用修辞、句式模式 |
| 人物关系图谱 | 主要角色及其相互关系 |
| 世界观体系 | 原作的世界设定和规则体系 |
| 情节骨架 | 主要情节节点和因果链 |
| 意象体系 | 作者常用的意象和隐喻 |
| 标志性表达 | 原作特有的语言标记（禁用词组候选） |

### 7.4 第二阶段：改写圣经

分析完成后，系统自动生成**改写圣经**（Rewrite Bible）。改写圣经是 AI 执行改写的核心指导文档，可以在生成后手动编辑。

**改写圣经包含：**

- **风格转化方案**：将原著写作风格转化为新风格的具体方案
- **角色改名方案**：原著角色名 → 改写后角色名的映射表
- **世界观转化方案**：核心元素到新设定框架的映射
- **情节抽象提炼**：情节结构化后再用新具体情节实例化的方案
- **禁止词组**：原著标志性短语，不得出现在改写版本中
- **意象迁移表**：原著标志性意象 → 新意象的替换映射

> **最佳实践**：生成改写圣经后，花 10-15 分钟仔细审查禁止词组和意象迁移表，手动删除泛化条目，添加 AI 漏掉的标志性表达。

### 7.5 第三阶段：改写章节

改写圣经确认后，点击「开始改写」（异步并发处理所有章节）。

**进度查看：**

改写任务面板中可以看到每个章节的改写状态：等待中 / 改写中 / 改写完成 / 改写失败。

### 7.6 查看改写结果

**对比视图：**

左右两栏分别显示原文和改写结果，差异高亮显示。可以对改写结果进行手动编辑，编辑后点击「应用」将改写结果更新到对应章节。

**合规报告：**

| 指标 | 说明 | 绿色阈值 |
|------|------|---------|
| 词法相似度 | 改写版本与原文的字词重叠比例 | < 20% |
| 结构相似度 | 段落结构重叠比例 | < 30% |
| 语义泄漏率 | 原作特有实体名称在改写中出现的比例 | < 10% |
| 质量分 | 改写文本的综合质量评估 | > 75 |

**总体评级：**

| 评级 | 条件 |
|------|------|
| 绿色（通过） | 通过率 ≥ 80% 且词法相似度 < 20% |
| 黄色（中间态） | 其他情况 |
| 红色（不通过） | 通过率 < 60% 或词法相似度 ≥ 35% |

### 7.7 重新改写与调整

对于合规报告中不通过的章节：

1. 点击章节右侧的「重新改写」，使用相同圣经重新执行改写
2. 先修改改写圣经（加强禁止词组，更换意象），再点击「重新改写」
3. 直接在改写结果文本编辑器中手动修改不通过的部分，修改后点击「重新评分」

## 8. 素材库

独立的媒体资源管理区域，可在多个小说和视频项目之间复用图片、音频、视频素材。

### 8.1 上传素材

进入「素材库」→ 点击「上传」，支持以下文件类型：

| 类型 | 支持格式 |
|------|----------|
| 图片 | JPG / PNG / WebP / HEIC |
| 音频 | MP3 / WAV / AAC / OGG |

上传后可以：

- 添加手动标签（如：山景、古典音乐、战斗场景）
- 点击「AI 自动打标签」，AI 识别内容并自动添加语义标签

- 添加评论和注释

### 8.2 搜索与过滤

**关键词搜索：**

在搜索框输入关键词，按类型（图片/音频/视频）过滤。

**语义搜索：**

输入描述性文字（如"夕阳下的古代城墙"），系统根据语义匹配最相关的素材，无需精确关键词匹配。

**过滤选项：**

- 按上传时间排序
- 按文件大小过滤
- 按标签筛选
- 仅显示公共库素材

### 8.3 素材集合

将相关素材组织为集合（Collection）：

1. 点击「新建集合」，填写名称和描述
2. 将素材拖入集合或点击「添加到集合」
3. 通过集合视图快速访问同一主题的素材组

### 8.4 版本管理

每个素材文件都支持版本管理：

1. 点击素材 → 「上传新版本」，选择替换文件
2. 历史版本保留，可以在「版本历史」中查看所有版本
3. 点击任意历史版本的「回滚」，将素材还原到该版本

### 8.5 分享与公共库

**申请分享到公共库：**

1. 点击素材 → 「申请分享到公共库」
2. 确认素材的版权说明（仅可分享自有版权或免版权素材）
3. 提交审核，管理员审核通过后素材对所有工作空间成员可见

**外链分享：**

点击「创建分享链接」，生成可直接访问的外链，支持设置过期时间和访问限制。

### 8.6 批量爬取

从免版权素材平台批量抓取并入库：

**创建爬取任务：**

1. 点击「爬取任务」→「新建爬取任务」
2. 选择来源平台
3. 输入关键词（支持多个关键词，逗号分隔）
4. 设置最大数量（建议单次不超过 100 条）
5. 点击「开始爬取」，后台自动抓取并入库

**垃圾桶与配额：**

- 删除的素材进入垃圾桶，30 天后自动永久删除，可在此期间恢复
- 在「账号配额」中查看当前存储使用量和上限

## 11. 常见问题

### Q1：章节生成质量不高怎么办？

按优先级逐步尝试：

1. **完善世界观和角色设定**：AI 上下文越丰富，生成质量越高
2. **在大纲中为本章添加详细提纲**：明确本章目标、场景顺序、高潮节点
3. **善用知识库**：将重要设定细节入库，生成时会自动检索注入上下文
4. **切换到更强的 AI 模型**：为章节生成任务切换到 GPT-4o 或 Claude
5. **调低 Temperature 参数**：降至 0.7-0.75，减少随机性，提高一致性
6. **生成后使用 AI 深度审查**：找出具体问题并应用修改建议

### Q2：视频分镜图片风格不一致怎么办？

1. **锁定场景参考图**：进入小说详情 → 「场景」标签，为出现频率最高的 3-5 个场景生成并锁定参考图
2. **生成角色三视图**：在「角色」标签中，为主要角色生成三视图
3. **为镜头绑定场景和角色**：在视频编辑器的分镜脚本中，为每个镜头指定对应的场景锚点和出场角色
4. **统一图像生成模型**：确保所有镜头使用同一个图像生成提供商和模型
5. **使用高清放大**：生成后在灯箱中使用「高清处理」提升图像质量，锐化边缘细节

### Q3：配音和画面时长不同步？

- 系统默认以**配音时长**为该镜头的实际播放时长
- 如果配音比设定时长更短，导出时镜头末尾会静音填充
- 如仍感觉不同步：
  1. 在配音标签中调节语速（降低语速会延长配音时长）
  2. 在分镜脚本中手动修改该镜头的「时长」字段
  3. 使用多段配音分段控制各段的节奏

### Q4：剪映打开后素材显示"离线"？

说明素材使用的 CDN 链接已过期，或网络无法访问到存储服务器。解决方法：

1. 使用「素材包（ZIP）」格式导出，获取本地化的图片和音频文件
2. 在剪映中对离线素材逐个点击「定位文件」，手动指向本地文件路径
3. 或直接删除离线素材并从本地重新导入

### Q5：大纲生成失败或一直转圈？

1. **检查 AI 模型配置**：确认大纲生成任务已在「任务映射」中分配模型，API Key 填写正确
2. **减少目标章节数**：超过 50 章的大纲生成可能触发超时，建议分批规划
3. **在任务面板查看错误详情**：任务面板（顶部导航的钟形图标）中可以看到具体的错误信息
4. **切换模型**：Claude 和 GPT-4o 的 JSON 格式稳定性较好

### Q6：任务面板里有任务一直"运行中"？

各类任务的参考完成时间：

- AI 文本生成：1-5 分钟
- 图片生成：30-90 秒
- 图片高清放大（算法）：10-30 秒
- 图片高清放大（AI）：1-3 分钟
- 视频生成（AI 视频模式）：5-15 分钟
- 小说改写（并发处理所有章节）：10-30 分钟

如超过以上时间仍未完成：刷新页面查看任务详情，如显示"失败"可点击「重新执行」。

### Q7：如何写中文小说但用英文 Prompt 生成分镜图片？

这是系统的默认行为，无需特殊设置：

- 正文和大纲：始终为中文
- 分镜的「画面描述」字段：AI 在生成分镜脚本时自动输出英文，直接用于图像生成
- 角色/物品/场景的描述：在「设置」标签中将「创作语言」改为「English」，AI 生成这些字段时会使用英文，进一步提升图像生成质量

### Q8：伏笔太多如何管理？

1. **按级别过滤**：在伏笔标签切换到「主线」视图，只关注最重要的伏笔
2. **关注超期预警**：优先处理橙色/红色预警的伏笔，安排回收或修改预计章节
3. **使用时间轴视图**：直观看到哪个阶段的伏笔密度过高
4. **添加强化记录**：对重要伏笔在中间章节手动添加「强化」记录，保持读者的记忆度
5. **适时废弃**：对于无法合理回收的低级别伏笔，可以手动标记废弃，保持列表整洁

### Q9：可以同时使用多个 AI 提供商吗？

可以，这正是推荐的使用方式：

- 为不同任务类型分配不同提供商：如大纲生成用 DeepSeek，AI 审查用 Claude，图片生成用可灵
- 多个提供商可以并行工作，互不干扰
- 在「任务映射」中，每种任务类型可以独立指定提供商和模型

### Q10：如何估算 API 费用？

**文本生成费用（以 DeepSeek 为例）：**

| 任务 | 估算 Token 消耗 | 参考费用（DeepSeek 单价） |
|------|----------------|--------------------------|
| 单章生成（2000字） | 约 3000 Token | 约 ¥0.003 |
| 100 章小说全部生成 | 约 30 万 Token | 约 ¥0.3 |
| AI 深度审查（单章） | 约 5000 Token | 约 ¥0.005 |

> 以上为粗略估算，实际消耗因上下文长度、层级摘要等因素有所差异。建议在提供商平台设置消费限额，避免超支。

**图片生成费用（以可灵为例）：**

- 每张图片约 ¥0.05-0.10
- 一个 30 镜头的视频项目约 ¥1.5-3.0
- 算法高清放大：免费（本地计算）
- AI 高清放大：约 ¥0.05-0.10 / 张

**视频生成费用（以可灵视频 API 为例）：**

- 每 5 秒视频约 ¥0.5-1.0
- 单集视频（30 镜头 × 5 秒）约 ¥15-30

### Q11：批量生成章节时出现角色遗忘怎么办？

简影使用**层级上下文记忆系统**来缓解长篇创作中的遗忘问题：

- 最近 2 章：完整内容注入上下文
- 最近 3-10 章：每章的简短摘要
- 每 10 章生成一次弧线摘要，记录角色状态和剧情重大节点
- 全局摘要：整部小说的核心设定

如仍出现遗忘，可以：

1. 确保角色描述和世界观设定在小说管理页面填写完整
2. 使用知识库记录重要设定细节（自动语义检索）
3. 在出现遗忘的章节前，手动使用「同步角色状态」功能
4. 在该章节的大纲中，手动写入角色的当前状态提醒

### Q12：能否为视频中的角色设置固定音色？

可以。在小说管理 → 「角色」标签中：

1. 点击角色卡，进入角色详情
2. 在「声线配置」区域选择该角色的专属音色（提供商 + 音色 ID）和语速
3. 保存后，视频配音阶段，含有该角色台词的镜头段落会自动使用对应音色
4. 在多段配音模式下，系统会自动将不同角色的台词分配到对应音色

> **提示**：建议为主角设置一个有辨识度的音色（如 MiniMax 的精英青年或御姐音），与旁白的音色明显区分，提升视频表现力。
